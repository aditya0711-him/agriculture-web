# Tests - Agriculture Web Application\n\n## Overview\nThis directory contains all test files for the Agriculture Web project.\n\n## Testing Strategy\n\n### Unit Tests\nTest individual functions and components in isolation.\n\n**Location:** `unit/`\n\n**Tools:**\n- Frontend: Jest, React Testing Library\n- Backend: Jest, Mocha, Chai\n\n### Integration Tests\nTest interactions between multiple components/services.\n\n**Location:** `integration/`\n\n**Tools:**\n- Supertest (API testing)\n- Jest\n- Postman Collections\n\n### End-to-End (E2E) Tests\nTest complete user workflows.\n\n**Location:** `e2e/`\n\n**Tools:**\n- Cypress\n- Selenium\n- Playwright\n\n## Directory Structure\n\n```\ntests/\n├── unit/\n│   ├── frontend/\n│   │   ├── components/\n│   │   └── utils/\n│   └── backend/\n│       ├── controllers/\n│       ├── models/\n│       └── services/\n├── integration/\n│   ├── api/\n│   └── database/\n├── e2e/\n│   ├── auth/\n│   ├── dashboard/\n│   └── crops/\n├── fixtures/\n│   └── mock-data.json\n└── config/\n    └── test-setup.js\n```\n\n## Running Tests\n\n### All Tests\n```bash\nnpm test\n```\n\n### Unit Tests Only\n```bash\nnpm run test:unit\n```\n\n### Integration Tests\n```bash\nnpm run test:integration\n```\n\n### E2E Tests\n```bash\nnpm run test:e2e\n```\n\n### With Coverage\n```bash\nnpm run test:coverage\n```\n\n## Test Examples\n\n### Unit Test (Frontend Component)\n```javascript\n// tests/unit/frontend/components/Dashboard.test.js\nimport React from 'react';\nimport { render, screen } from '@testing-library/react';\nimport Dashboard from '../../../../frontend/src/components/Dashboard';\n\ndescribe('Dashboard Component', () => {\n  test('renders dashboard title', () => {\n    render(<Dashboard />);\n    expect(screen.getByText(/Dashboard/i)).toBeInTheDocument();\n  });\n});\n```\n\n### Integration Test (API)\n```javascript\n// tests/integration/api/auth.test.js\nconst request = require('supertest');\nconst app = require('../../../backend/src/app');\n\ndescribe('Authentication API', () => {\n  test('POST /api/auth/register - should register new user', async () => {\n    const response = await request(app)\n      .post('/api/auth/register')\n      .send({\n        username: 'testuser',\n        email: 'test@example.com',\n        password: 'password123'\n      });\n    \n    expect(response.status).toBe(201);\n  });\n});\n```\n\n### E2E Test\n```javascript\n// tests/e2e/auth/login.test.js\ndescribe('Login Flow', () => {\n  beforeEach(() => {\n    cy.visit('http://localhost:3000/login');\n  });\n\n  it('should successfully login with valid credentials', () => {\n    cy.get('input[name=\"email\"]').type('user@example.com');\n    cy.get('input[name=\"password\"]').type('password123');\n    cy.get('button[type=\"submit\"]').click();\n    cy.url().should('include', '/dashboard');\n  });\n});\n```\n\n## Coverage Goals\n\n- **Overall Coverage:** > 80%\n- **Critical Paths:** 100%\n- **Components:** > 90%\n- **API Endpoints:** > 95%\n\n## CI/CD Integration\n\nTests run automatically on:\n- Pull requests to `develop` and `main`\n- Commits to `main` before production deployment\n\n## Contributing\n\nWhen adding new features:\n1. Write tests first (TDD)\n2. Ensure all tests pass\n3. Maintain coverage above 80%\n4. Include both positive and negative test cases\n\nSee CONTRIBUTING.md in the root directory.\n"
